VFK

VFK driver can read data in the Czech cadastral exchange data format. Each VFK file is represented as one datasource. A VFK file may have 0, 1 or more layers.

Points are represented as wkbPoint, lines and boundaries as wkbLineString and areas as wkbPolygon. wkbMulti* and are not used. Feature types cannot be mixed in one layer.

Datasource name

Datasource name is full path to VFK file.

Layer names

VFK data blocks are used as layer names.

Attribute filter

OGR internal SQL engine is used to evaluate the expression. Evaluation is done once when the attribute filter is set.

Spatial filter

Bounding boxes of features stored in topology structure are used to evaluate if a features matches current spatial filter.

Evaluation is done once when the spatial filter is set.

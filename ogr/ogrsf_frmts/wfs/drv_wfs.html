WFS - OGC WFS service

(GDAL/OGR >= 1.8.0)

This driver can connect to a OGC WFS server. It supports WFS 1.0.0 and WFS 1.1.0 protocols. Curl must be configured for the driver to be returned. Usually WFS return results in GML format, so the GML driver should generally be set-up for read-only (configured with Xerces or Expat support). It is possible to use alternate underlying formats when the server supports them (such as OUTPUTFORMAT=json).

The driver is currently read-only (no WFS-T).

The minimal syntax to open a WFS datasource is : WFS:http://path/to/WFS/service or http://path/to/WFS/service?SERVICE=WFS

Additionnal optional parameters can be specified such as TYPENAME or VERSION, as specified in WFS specification.

Examples

  • Listing the types of a WFS server : 
    ogrinfo -ro WFS:http://www2.dmsolutions.ca/cgi-bin/mswfs_gmap

  • Listing the types of the popplace layer with a spatial filter : 
    ogrinfo -ro WFS:http://www2.dmsolutions.ca/cgi-bin/mswfs_gmap popplace -spat 0 0 2961766.250000 3798856.750000

Oracle Spatial GeoRaster (DRAFT)

This driver supports reading and writing data in Oracle Spatial (10g or later) Object-Relational format. The Oracle Spatial GeoRaster driver is not normally built into GDAL, but may be built in on platforms where the Oracle client libraries are available.

When opening a database, it's name should be specified in the form:

georaster:<user>,<pwd>,[db],[table],[column],[where]

or

georaster:<user>,<pwd>,[db],<rdt>,<rid>

Where:

user = Oracle server user's name login

pwd = user passord

db = Oracle server identification (optional, uses user's default)

table = name of a GeoRaster table (table that contains GeoRaster columns)

column = name of a table's column wich data type is MDSYS.SDO_GEORASTER

where = a simple where clause to identify one ore multiples GeoRaster

rtd = name of an raster data table

rid = numeric identification of one GeoRaster (must be unique on rtd)

The separator characters could be one of ",/@:" in combined use.

Examples:

geor:scott,tiger,demodb,table,column,id=1

"georaster:scott/tiger@demodb,table,column,gain>10"

"georaster:scott/tiger@demodb,table,column,city=london"

georaster:scott,tiger,,rdt_10$,10

geor:scott/tiger,,rdt_10$,10

Creation Options

  • OVERWRITE: This may be "YES" or "ON" to alow existing GeoRaster to be overwritten

  • .BLOCKXSIZE: The number of pixel columns on raster block.

  • .BLOCKYSIZE: The number of pixel rows on raster block.

  • .BLOCKBSIZE: The number of bands on raster block.

  • .INTERLEAVE: Band interleaving mode, BAND, LINE, PIXEL for band sequential, Line or Pixel interleaving. Note that Oracle Spatial accepts LINE and PIXEL interleaving mode for single band raster datasets.

  • .DESCRIPTION: A simple description of a newly created table in SQL sintax. If the table already exist, this create option will be ignored.

  • .INSERT: A simple SQL insert/values clause to informe the driver what values to fill up when inserting a new row on the table.
